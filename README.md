# Gaia--HST validation workflow for globular-cluster proper-motion dispersion profiles

This repository contains the MATLAB workflow, input tables, archived output CSV files, and metadata needed to reproduce the Gaia--HST profile-level validation described in the manuscript:

**A Reproducible Gaia--HST Internal-Kinematics Validation Pipeline for Galactic Globular Clusters**

The workflow compares Gaia EDR3 radial proper-motion dispersion profiles with HST/HACKS proper-motion dispersion profiles in the common radial domain and stores cluster-level validation classes. APOGEE DR17 globular-cluster information is carried only as a spectroscopic-readiness layer.

## Repository contents

```text
src/                 MATLAB source functions
scripts/             MATLAB run/check scripts and a Python CSV-output check
input/               Input CSV tables and raw HACKS text used by the workflow
output_csv/          Archived machine-readable output tables used by the manuscript
metadata/            File manifest, column dictionary, source provenance, checksums, and audit tables
docs/                Data/code availability text and release instructions
environment/         Runtime environment notes
figures/             Placeholder for figures generated by the MATLAB workflow
logs/                Placeholder for run logs generated by the MATLAB workflow
```

## Main archived output tables

The output tables in `output_csv/` are the machine-readable products used to support the manuscript statistics:

- `GaiaHST_MATCHED_RADIAL_COMPARISON.csv`
- `GaiaHST_CLUSTER_QUALITY_CLASSES.csv`
- `GaiaHST_GLOBAL_CALIBRATION_SUMMARY.csv`
- `GaiaHST_DATA_LAYER_READINESS.csv`
- `GaiaHST_ROBUSTNESS_SUMMARY.csv`
- `GaiaHST_BOOTSTRAP_GLOBAL_MEAN.csv`
- `GaiaHST_CLASS_BOUNDARY_SENSITIVITY.csv`

Additional summary CSV files are included for manuscript tables and repository-level checks.

## Reproduction in MATLAB

From the repository root, run:

```matlab
addpath('src')
run('scripts/run_gaia_hst_validation.m')
```

The workflow reads files from `input/` and writes generated products under `outputs/`.

To regenerate the HST/HACKS CSV from the raw text table before running the analysis, run:

```matlab
addpath('src')
run('scripts/run_with_hacks_conversion.m')
```

## Output check without MATLAB

The archived output CSV files can be checked with:

```bash
python3 scripts/check_csv_outputs.py
```

Expected principal values:

- matched radial bins: 680
- matched clusters: 56
- global mean fractional residual: approximately -0.0028
- global median fractional residual: approximately -0.0010
- global RMS fractional residual: approximately 0.1322
- class counts: A=38, B=8, C=8, D=2, E=0

## Metadata

- `metadata/file_manifest.csv`: row counts, column counts, and role of each CSV file.
- `metadata/column_dictionary.csv`: inferred column types, units, and missing-value policy.
- `metadata/source_provenance.csv`: data-layer provenance and processing notes.
- `metadata/sample_scope_notes.csv`: notes on records outside the globular-cluster validation sample.
- `metadata/reproducibility_audit.csv`: principal checks for the archived results.
- `metadata/file_checksums_sha256.csv`: SHA256 checksums for released files.

## Data-use scope

The archived output validates profile-level products in the Gaia--HST radial-overlap domain. The comparison does not rederive individual-star Gaia or HST proper motions and does not impose magnitude-matched or population-matched selections. APOGEE fields are metadata for future spectroscopic validation and are not used in the baseline Gaia--HST residual calculation.
