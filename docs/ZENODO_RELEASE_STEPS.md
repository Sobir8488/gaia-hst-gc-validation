# GitHub--Zenodo release steps

1. Create a public GitHub repository.
2. Upload the contents of this release directory to the repository root.
3. Commit the files and push to GitHub.
4. Log in to Zenodo with GitHub and enable archiving for the repository.
5. Create a GitHub release, for example `v1.0.0`.
6. Zenodo archives the release and mints a DOI.
7. Add the DOI to the manuscript data/code availability statement, repository README, and citation text.

Do not upload duplicate zip files inside the repository. The GitHub release itself will be archived by Zenodo.
